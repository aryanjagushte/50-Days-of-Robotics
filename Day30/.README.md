## What is URDF?

URDF (Unified Robot Description Format) is the standard format for describing robots in ROS systems. A URDF file defines:

- **Links**: rigid parts of the robot (body, arms, wheels, etc.)
- **Joints**: connections between links that allow movement
- **Geometry**: 3D shapes of each link
- **Material properties**: colors and textures
- **Inertial properties**: mass, center of mass, and inertia matrix

For ROS 2 to simulate or control a robot, it must have a valid URDF file.

---

## URDF Structure

A URDF file is written in XML format and follows a specific structure:

```xml
<robot name="robot_name">
  <link name="link_name">
    <!-- geometry, material, inertial properties -->
  </link>
  
  <joint name="joint_name" type="joint_type">
    <!-- parent, child, axis, limits -->
  </joint>
  
  <!-- Gazebo-specific tags -->
  <gazebo>
    <!-- simulation parameters -->
  </gazebo>
</robot>
```

This structure defines the complete robot model.

---

## Creating Links in URDF

Links are the rigid parts of the robot. Each link needs:

- **name**: unique identifier for the link
- **geometry**: 3D shape (box, cylinder, sphere, mesh)
- **material**: color and appearance
- **inertial**: mass and inertia properties

Common geometry types include:
- `<box>`: rectangular shapes
- `<cylinder>`: cylindrical parts
- `<sphere>`: round objects
- `<mesh>`: 3D models from CAD files

---

## Creating Joints in URDF

Joints connect links and define how they move relative to each other.

Each joint needs:
- **name**: unique identifier
- **type**: revolute, continuous, fixed, prismatic, etc.
- **parent**: the link this joint connects to
- **child**: the link this joint connects from
- **axis**: direction of movement
- **limits**: range of motion (for revolute joints)

Common joint types:
- **revolute**: rotates within limits
- **continuous**: rotates without limits (like wheels)
- **fixed**: no movement between links
- **prismatic**: slides along an axis

---

## Gazebo-Specific Tags

For URDF to work properly in Gazebo, you need to add Gazebo-specific tags:

```xml
<gazebo reference="link_name">
  <material>Gazebo/MaterialName</material>
  <mu1>friction_value</mu1>
  <mu2>friction_value</mu2>
</gazebo>
```

These tags define:
- visual materials for Gazebo
- friction properties
- collision properties
- physics parameters

Without these tags, Gazebo may not simulate the robot correctly.

---

## Adding Physics Properties

Physics properties are essential for realistic simulation:

- **Mass**: affects how the robot moves
- **Inertia**: affects rotational dynamics
- **Friction**: affects how surfaces interact
- **Damping**: affects motion resistance

These properties must be set correctly for the robot to behave realistically in Gazebo.

---

## Sensor Simulation in URDF

URDF can also define sensors for Gazebo simulation:

```xml
<joint name="sensor_joint">
  <!-- sensor attachment -->
</joint>

<gazebo reference="sensor_link">
  <sensor name="sensor_name" type="sensor_type">
    <!-- sensor configuration -->
  </sensor>
</gazebo>
```

Common sensors include:
- **camera**: visual imaging
- **depth**: distance measurement
- **IMU**: orientation and acceleration
- **lidar**: 2D/3D scanning
- **gps**: positioning

---

## Testing URDF in ROS 2

After creating the URDF file, you need to test it in ROS 2:

1. **Load in RViz**: visualize the robot model
2. **Load in Gazebo**: simulate with physics
3. **Check for errors**: verify URDF is valid
4. **Test movement**: confirm joints work correctly
5. **Test sensors**: verify sensor data is generated

This testing ensures the URDF model is simulation-ready.
