# Day 32 - Gazebo in ROS 2: Installing and Spawning Your Robot

## Overview

Day 32 explains **what Gazebo is**, how to **install Gazebo for ROS 2 Humble**, and how to **spawn your Fusion 360–based robot into Gazebo Classic** using the launch file from the `squarebot_main_description` package. It also briefly introduces **Gazebo Harmonic** and explains the difference between Gazebo Classic and the newer Gazebo Sim series.[page:0]

This day is about getting from “robot designed and converted to URDF” to “robot running in a 3D physics simulator,” which is a critical step before moving to real-world testing.[page:0]

---

## What is Gazebo?

Gazebo is a **3D physics simulator** used to test robots in a virtual environment before deploying them to hardware.[page:0]

Key points from the video:[page:0]

- Gazebo discussed here is **Gazebo Classic**, which is now **end-of-life (EOL)**.[page:0]  
- There is a newer series called **Gazebo Sim**, which includes **Gazebo Harmonic**.[page:0]  
- Gazebo is specifically designed to integrate with the **Robot Operating System (ROS/ROS 2)**.[page:0]

The instructor also compares Gazebo to other 3D tools like Blender or Webots but emphasizes that Gazebo is built for robotics and ROS integration.[page:0]

---

## Installing Gazebo for ROS 2 Humble

The video shows how to install Gazebo and related ROS 2 packages on **ROS 2 Humble**.[page:0]

### Core Gazebo installation

```bash
sudo apt install ros-humble-gazebo*
```

- The wildcard `*` ensures all relevant Gazebo-related packages for Humble are installed.[page:0]

### Additional useful packages

The instructor also installs supporting packages (for extensions and plugins):[page:0]

```bash
sudo apt install ros-humble-gazebo-ros-pkgs*
sudo apt install ros-humble-gazebo-plugins*
sudo apt install ros-humble-ros2-control*   # shown as 'robot' family in the video
```

He mentions that his system already has many of these installed, so the package manager offers only updates, but for a fresh system you should allow these to install.[page:0]

The key message: **make sure these Gazebo and ROS 2 integration packages are installed before trying to simulate your robot.**[page:0]

## Gazebo Classic vs Gazebo Harmonic

The video briefly introduces the two Gazebo families:[page:0]

- **Gazebo Classic**  
  - The version used in this video.  
  - End-of-life (EOL) but still widely used in many tutorials.[page:0]

- **Gazebo Sim / Gazebo Harmonic**  
  - Part of the newer Gazebo line known as Gazebo Sim.[page:0]  
  - The instructor shows that Harmonic exists but focuses the demo on **Gazebo Classic** for this session.[page:0]

This helps learners understand that there are two generations of Gazebo and that this day centers on **Classic**.

## Opening Gazebo Classic and Exploring the UI

Before spawning the robot via launch files, the instructor opens **Gazebo Classic** directly to show the basic environment.[page:0]

Key observations:[page:0]

- Gazebo provides a 3D world with **X, Y, Z axes**, similar to Fusion 360.[page:0]  
- You **cannot design** robots directly in Gazebo; it is a *simulation* environment, not a CAD tool.[page:0]  
- You can:
  - Add primitive shapes  
  - Change lighting (spot light, point light, directional light)  
  - Scale shapes in the scene using the scale tool[page:0]

However, the main focus is **simulating robots**, not building geometry from scratch.[page:0]

## Spawning the Robot in Gazebo Using ROS 2

The core part of the video shows how to spawn the Fusion 360–based robot into Gazebo Classic using ROS 2.[page:0]

### Steps followed

1. **Open terminal** and go to the mobile robot workspace:[page:0]

```bash
cd ~/mobile_robot_ws
```

2. **Source the workspace:**

```bash
source install/setup.bash
```

3. **Run the Gazebo launch file:**

```bash
ros2 launch squarebot_main_description gazebo.launch.py
```

- The instructor zooms the terminal and shows the command clearly.[page:0]  
- After a short wait, **Gazebo opens and the robot spawns automatically in the world**.[page:0]

He explicitly mentions that **this is the same robot that was designed in Fusion 360**, now simulated inside Gazebo Classic.[page:0]

---

## What the Launch File Does

The Gazebo launch file (`gazebo.launch.py`) handles:[page:0]

- Starting Gazebo Classic  
- Loading the robot’s URDF description  
- Spawning the robot into the world with proper plugins and dependencies[page:0]

The instructor suggests opening this file in VS Code:[page:0]

- Go to the package  
- Open `launch/gazebo.launch.py`  
- Inspect the dependencies listed there (e.g., Gazebo, robot state publisher, spawn entity, etc.)[page:0]

This helps learners understand how launch files tie together URDF, Gazebo, and ROS 2 nodes.
