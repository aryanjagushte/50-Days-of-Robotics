# Day 14 - ROS 2 Architecture and Workspace Creation

## Overview

Day 14 begins the practical understanding of **ROS 2 architecture** by introducing one of its most important foundational concepts: the **workspace**. Instead of trying to learn the complete ROS 2 architecture in a single day, this lesson breaks the topic into smaller and easier parts so that beginners can understand the system step by step.

This day mainly focuses on the **file system side of ROS 2**, especially the idea of a workspace and how to create one properly. It also gives a brief introduction to the second side of ROS 2 architecture, called the **computational graph**, which includes concepts like nodes, topics, messages, and services that will be covered in later days.

---

## What is ROS 2 Architecture?

ROS 2 architecture can be understood as the structure that helps organize a robotics project in a clean and manageable way.

According to this lesson, ROS 2 architecture can be broadly divided into **two main parts**:

1. **File System Part**
   - Workspaces
   - Packages

2. **Computational Graph Part**
   - Nodes
   - Topics
   - Messages
   - Services

This day starts with the **workspace** because it is easier to understand first and creates the foundation needed before moving to more advanced ROS 2 communication concepts.

---

## Why ROS 2 Architecture is Important

As robotics projects become bigger, the code and files also become more complex. If everything is placed in random folders or managed without structure, the project becomes hard to maintain, update, and scale.

ROS 2 architecture helps solve this problem by:
- Organizing robotics projects properly
- Separating different parts of the robot into manageable units
- Making it easier to update code and packages
- Reducing project complexity
- Preparing the system for scalable robotics development

This is why understanding architecture is not just theory—it is directly connected to how real robotics projects are built.

---

## What is a Workspace in ROS 2?

**Simple Definition:**

A **workspace** is the environment or folder where you manage your ROS 2 project files, packages, and related development work.

You can think of a workspace as:
- The main folder of your robotics project
- The place where different packages are stored
- The organized space where development happens

The video explains this in a very beginner-friendly way:  
a workspace is simply the place where you keep and manage all the files related to your project.

---

## Why Workspaces Matter

Workspaces are important because robotics projects usually contain multiple packages and modules. Instead of keeping everything together in a confusing way, workspaces help organize them properly.

### Benefits of Using Workspaces

- Easier project management
- Better organization of files and packages
- Reduced complexity
- Easier modification of code later
- Better structure for different robot types and projects

For example, you may create separate workspaces for:
- Mobile robots
- Robotic arms
- Drones
- Core robot software
- Different experiments or simulations

This makes development cleaner and easier to manage.

---

## Workspace and Packages Relationship

A very important point from Day 14 is that the workspace is not the package itself. Instead, the workspace is the **place where packages are maintained**.

### Simple Relationship

- **Workspace** = project environment / main folder
- **Packages** = individual components inside the workspace

This idea is important because later in ROS 2, you will create multiple packages inside a workspace, and each package may handle a specific part of the robot such as:
- Sensor processing
- Motor control
- Navigation
- Vision
- Communication

So Day 14 gives the first step toward that structured development process.

## File System Part of ROS 2

This lesson focuses specifically on the **file system part** of ROS 2 architecture.

The file system part includes:
- Workspace
- `src` folder
- Packages inside `src`

The video mentions that there is a kind of file system structure that needs to be followed in ROS 2. This is important because ROS 2 expects projects to follow a standard format so tools and commands can work properly.

---

## Creating a ROS 2 Workspace

In the practical part of the lesson, a workspace is created for a mobile robot project.

### Example Workspace Name

```bash
mkdir mobile_robot
```

This creates the main folder for the ROS 2 workspace.

### Move Inside the Workspace

```bash
cd mobile_robot
```

After entering the workspace, the next important step is creating the `src` folder.

### Create the `src` Folder

```bash
mkdir src
```

This is one of the most important steps in the lesson.

---

## Why the `src` Folder is Important

The `src` folder is the standard place inside a ROS 2 workspace where packages are created and managed.

### Why `src` Matters

- Keeps the project structured
- Acts as the main location for ROS 2 packages
- Follows the expected ROS 2 file system format
- Makes future package creation possible

So the minimum basic structure introduced in Day 14 is:

```bash
mobile_robot/
└── src/
```

This simple structure becomes the foundation for future ROS 2 package development.

---

## Practical Command Flow

Here is the practical workflow covered in the video:

```bash
mkdir mobile_robot
cd mobile_robot
mkdir src
```

After this, you have a basic ROS 2 workspace structure ready for future development.
