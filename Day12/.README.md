# Day 12 - Basic Ubuntu Commands for ROS 2 Beginners

## Overview

Day 12 introduces the **basic Ubuntu commands** every beginner should know before moving deeper into ROS 2 and robotics development. Since ROS 2 is commonly used on **Ubuntu 22.04**, understanding how to work with the Linux terminal is a very important skill for writing commands, managing files, and handling development workflows.[page:4]

This day focuses on practical terminal usage for complete beginners. The main goal is to become comfortable with the Ubuntu terminal and learn the basic commands that will be used repeatedly throughout the ROS 2 journey.[page:4]

## Why Ubuntu Commands Matter in Robotics

When working with ROS 2, you will spend a lot of time in the terminal. Installation, package setup, workspace creation, running nodes, building projects, and troubleshooting often happen through command-line tools rather than only through graphical interfaces.[page:4]

Learning Ubuntu commands helps you:
- Work efficiently in the Linux environment.
- Understand ROS 2 installation and execution steps.
- Navigate folders and files quickly.
- Manage packages and dependencies.
- Become comfortable with real robotics development workflows.[page:4]

## Using Terminator

Before starting with commands, Day 12 introduces **Terminator**, a terminal application that allows you to work with multiple terminal windows in a single interface. This is especially useful for ROS 2 because robotics workflows often require running several commands at the same time.[page:4]

### Why Terminator is Useful

- Lets you open multiple terminals in one window.
- Helps manage different ROS 2 processes more easily.
- Makes development more organized.
- Useful when running multiple nodes or commands together.[page:4]

### Installation Command

```bash
sudo apt install terminator
```

### Shortcut to Open Terminator

```bash
Ctrl + Alt + T
```

You can also split Terminator horizontally and vertically to run multiple terminal sessions side by side.[page:4]

## System Update Commands

These are some of the most common commands used in Ubuntu.

### 1. `sudo`

**Simple Meaning:**  
`sudo` gives administrator permission to run commands.

It is commonly used when installing software, updating packages, or making system-level changes.[page:4]

### 2. `sudo apt update`

This command updates the package list and checks what packages can be upgraded.[page:4]

```bash
sudo apt update
```

### 3. `sudo apt upgrade`

This command upgrades the packages that have available updates.[page:4]

```bash
sudo apt upgrade
```

These commands are very important because they help keep the Ubuntu system updated and ready for software installation.[page:4]

## File and Directory Navigation Commands

### 1. `ls`

The `ls` command is used to list the files and folders in the current directory.[page:4]

```bash
ls
```

**Why it matters:**
- Helps you see available files and folders.
- Useful for checking package folders, workspaces, and project files.[page:4]

### 2. `pwd`

The `pwd` command shows the path of the current directory you are working in.[page:4]

```bash
pwd
```

**Why it matters:**
- Helps you know exactly where you are in the terminal.
- Avoids confusion while navigating large project directories.[page:4]

### 3. `mkdir`

The `mkdir` command is used to create a new directory (folder).[page:4]

```bash
mkdir 50DaysOfRobotics
```

**Why it matters:**
- Helps create project folders and ROS workspaces.
- Useful for organizing files properly.[page:4]

### 4. `cd`

The `cd` command means **change directory**. It is used to move into a folder or move out of one.[page:4]

```bash
cd 50DaysOfRobotics
```

To go one directory back:

```bash
cd ..
```

**Why it matters:**
- Essential for moving between folders in Ubuntu.
- Very important when working with workspaces and packages in ROS 2.[page:4]

---

## File Creation and Management Commands

### 1. `touch`

The `touch` command is used to create a new file.[page:4]

```bash
touch day12.txt
```

**Why it matters:**
- Helps create text files quickly.
- Useful for notes, scripts, configuration files, and testing purposes.[page:4]

### 2. `cp`

The `cp` command is used to copy files from one location to another.[page:4]

```bash
cp day12.txt src/
```

**Why it matters:**
- Useful for duplicating files.
- Helps move data into project folders or package directories.[page:4]

### 3. `mv`

The `mv` command is used to move files from one directory to another. It can also be used to rename files.[page:4]

```bash
mv day12.txt src/
```

**Why it matters:**
- Useful for organizing files.
- Commonly used while managing ROS packages, scripts, and workspace content.[page:4]

---

## Deleting Files and Folders

### 1. `rm`

The `rm` command is used to delete files.[page:4]

```bash
rm day12.txt
```

**Why it matters:**
- Helps remove unwanted files.
- Useful during cleanup and testing.[page:4]

### 2. `rm -r`

The `rm -r` command is used to delete folders recursively.[page:4]

```bash
rm -r src
```

**Why it matters:**
- Helps remove folders and their contents.
- Useful when cleaning project directories or deleting unwanted folders.[page:4]

**Important Note:**  
Be careful while using `rm` and `rm -r` because deleted files and folders are not easily recovered.[page:4]

---

## Useful Terminal Shortcut

### Tab Autocomplete

A very useful shortcut introduced in this day is the **Tab key** for autocomplete.[page:4]

**How it helps:**
- Auto-completes file and folder names.
- Saves time while typing long names.
- Helps avoid typing mistakes.

For example:
- Type the first few letters of a file or folder.
- Press `Tab`.
- Ubuntu will try to auto-complete the name.[page:4]

If there are multiple matching options, pressing `Tab` twice can show suggestions.[page:4]

## Practical Example Workflow

Here is a simple command sequence based on the lesson:

```bash
sudo apt update
sudo apt upgrade
mkdir 50DaysOfRobotics
cd 50DaysOfRobotics
mkdir src
touch day12.txt
cp day12.txt src/
mv day12.txt src/
cd src
ls
rm day12.txt
cd ..
rm -r src
```

This kind of workflow helps beginners understand how basic commands connect together in real usage.[page:4]
