## What is SLAM?

**SLAM (Simultaneous Localization and Mapping)** is the process where a robot:

- **Builds a map** of an unknown environment
- **Locates itself** within that map
- Does both tasks **simultaneously**
- Uses sensors like lidar, cameras, or depth sensors
- Creates a representation of the environment
- Tracks its position as it moves

SLAM is essential because robots need to know both where they are and what the environment looks like to navigate autonomously.

---

## What is Navigation in ROS 2?

**Navigation in ROS 2** refers to the stack of tools that enables robots to:

- Plan paths from start to goal
- Avoid obstacles dynamically
- Follow planned paths
- Reach destinations autonomously
- Handle new maps and environments
- Integrate with SLAM for mapping

The ROS 2 Navigation Stack is a standard framework for robot navigation.

---

## How SLAM Enables Autonomous Movement

SLAM enables autonomous movement by:

- Creating a map of the environment the robot explores
- Allowing the robot to know its position in that map
- Providing the map data needed for navigation
- Enabling the robot to explore unknown spaces
- Supporting path planning using the generated map
- Letting the robot return to previously visited locations

Without SLAM, robots cannot navigate environments they haven't been mapped before.

---

## How Navigation Enables Autonomous Movement

Navigation enables autonomous movement by:

- Taking a goal position from the user
- Planning a path to that goal using the map
- Avoiding obstacles along the path
- Controlling the robot's motors to follow the path
- Adjusting the path if new obstacles appear
- Reaching the goal autonomously

Without Navigation, robots cannot move toward goals or avoid obstacles.

---

## The Navigation Stack in ROS 2

The ROS 2 Navigation Stack includes several components:

| Component | Purpose |
|-----------|---------|
| **SLAM** | Build map and localize robot |
| **Localization** | Track robot position in map |
| **Path Planning** | Generate path to goal |
| **Costmap** | Represent obstacles and free space |
| **Controller** | Execute path following |
| **Recovery** | Handle stuck situations |

These components work together to enable autonomous navigation.

## Sensor Requirements for SLAM and Navigation

SLAM and Navigation require specific sensors:

| Sensor Type | Purpose |
|-------------|---------|
| **Lidar** | Primary sensor for 2D/3D mapping |
| **Depth Camera** | Alternative for mapping and obstacle detection |
| **IMU** | Helps with orientation and localization |
| **Wheel Odometry** | Tracks movement for localization |
| **Camera** | Visual SLAM and obstacle recognition |

Having the right sensors is essential for SLAM and Navigation to work.
