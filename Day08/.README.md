## What is Computer Vision?

**Simple Definition:**

Computer vision is the field of AI that studies images and video to understand the visual world using cameras.

**Technical Definition:**

Computer vision is a subset of artificial intelligence that deals with extracting information from visual data (images or videos) captured by cameras, and using that information to make decisions or control systems.

**Why Computer Vision for Robotics?**

- **Perception:** Allows robots to see and interpret their environment.
- **Autonomy:** Enables robots to navigate, avoid obstacles, and follow paths.
- **Interaction:** Lets robots detect and recognize humans, objects, and gestures.
- **Task Execution:** Supports tasks like picking objects, sorting, and assembly.
- **Navigation:** Helps robots understand where they are and where to go.
- **AI Integration:** Bridges robotics with advanced AI and machine learning.

**Core Vision of Computer Vision:**

Instead of just capturing images, computer vision:
- Processes images and video.
- Extracts meaningful features (edges, colors, shapes).
- Detects objects and patterns.
- Makes decisions based on visual data.

## Computer Vision Basics

### 1. Images and Cameras

**What are Images?**

Images are 2D grids of pixels, where each pixel holds color or intensity information.

**Simple Definition:**
- Digital representation of a visual scene.
- Made up of pixels arranged in rows and columns.
- Each pixel has color values (e.g., RGB).

**In Computer Vision:**
- Images are the primary data source.
- Cameras capture these images in real time.
- Robots use these images to understand their surroundings.

**Image Representation:**

```python

```

**Why Images Matter in Robotics:**
- Provide visual data for navigation.
- Enable object detection and tracking.
- Support line following and color-based decisions.
- Help robots interact with humans and environments.

### 2. Video Processing

**What is Video?**

Video is a sequence of images (frames) captured over time.

**Simple Definition:**
- Multiple images shown in rapid succession.
- Each image is called a frame.
- Frames are captured at a specific rate (e.g., 30 frames per second).

**In Computer Vision:**
- Video allows robots to see motion.
- Robots can track objects as they move.
- Enables tasks like following a person or avoiding moving obstacles.

**Reading Video with OpenCV:**

```python
import cv2

# Open default camera
cap = cv2.VideoCapture(0)

while True:
    # Read a frame from the camera
    ret, frame = cap.read()
    
    if not ret:
        break
    
    # Show the frame
    cv2.imshow('Robot Camera', frame)
    
    # Exit on 'q' key press
    if cv2.waitKey(1) & 0xFF == ord('q'):
        break

cap.release()
cv2.destroyAllWindows()
```

**Why Video Matters in Robotics:**
- Detect moving objects and people.
- Track paths and trajectories.
- Monitor dynamic environments.
- Support real-time decision making.

### 3. Image Processing

**What is Image Processing?**

Image processing is the study of images and applying algorithms to modify or analyze them.

**Simple Definition:**
- Applying operations on images to extract information.
- Includes filtering, color changes, edge detection, etc.

**Common Image Processing Tasks:**
- **Filtering:** Remove noise, blur, or sharpen.
- **Color Conversion:** Convert between RGB, grayscale, HSV.
- **Edge Detection:** Find boundaries in images.
- **Thresholding:** Separate objects from background.
- **Feature Extraction:** Identify shapes, lines, corners.

**Basic Image Processing with OpenCV:**

```python
import cv2

# Load an image
image = cv2.imread('robot.jpg')

# Convert to grayscale
gray = cv2.cvtColor(image, cv2.COLOR_BGR2GRAY)

# Apply edge detection
edges = cv2.Canny(gray, 100, 200)

# Display the result
cv2.imshow('Edges', edges)
cv2.waitKey(0)
cv2.destroyAllWindows()
```

**Why Image Processing Matters in Robotics:**
- Clean up noisy sensor images.
- Detect edges for navigation.
- Identify colored objects (e.g., red line, blue ball).
- Prepare images for object detection and AI models.

## OpenCV: Open Source Computer Vision Library

**What is OpenCV?**

OpenCV is an open-source library that provides tools for computer vision and image processing.

**Simple Definition:**
- A Python library for working with images and video.
- Contains hundreds of functions for vision tasks.
- Widely used in robotics, AI, and research.

**Why OpenCV for Robotics?**

- **Easy to Use:** Simple Python API.
- **Fast:** Optimized for real-time performance.
- **Rich Features:** Includes object detection, tracking, feature extraction.
- **Cross-Platform:** Works on Windows, Linux, macOS, Raspberry Pi.
- **Community Support:** Large user base and tutorials.

## Practical Robotics Applications of Computer Vision

### 1. Line Following Robot

A robot that follows a colored line (e.g., black or red) using camera input.

```python
import cv2
import numpy as np

# Open camera
cap = cv2.VideoCapture(0)

while True:
    ret, frame = cap.read()
    if not ret:
        break
    
    # Convert to HSV color space
    hsv = cv2.cvtColor(frame, cv2.COLOR_BGR2HSV)
    
    # Define range for red color
    lower_red = np.array()
    upper_red = np.array()[2]
    
    # Threshold the image to get red color
    mask = cv2.inRange(hsv, lower_red, upper_red)
    
    # Find the center of the red line
    pixels = cv2.findNonZero(mask)
    if pixels is not None:
        x, y, w, h = cv2.boundingRect(pixels)
        center_x = x + w // 2
        
        # Decide movement based on center position
        if center_x < 300:
            print("Turn left")
        elif center_x > 340:
            print("Turn right")
        else:
            print("Move forward")
    
    # Show the mask
    cv2.imshow('Mask', mask)
    
    if cv2.waitKey(1) & 0xFF == ord('q'):
        break

cap.release()
cv2.destroyAllWindows()
```

**Why Line Following is Important:**
- Common in warehouse robots.
- Used in autonomous guided vehicles (AGVs).
- Simple way to test vision-based navigation.

### 2. Object Detection Robot

A robot that detects and tracks objects (e.g., balls, people).

```python
import cv2

cap = cv2.VideoCapture(0)

while True:
    ret, frame = cap.read()
    if not ret:
        break
    
    # Convert to grayscale
    gray = cv2.cvtColor(frame, cv2.COLOR_BGR2GRAY)
    
    # Edge detection
    edges = cv2.Canny(gray, 100, 200)
    
    # Find contours
    contours, _ = cv2.findContours(edges, cv2.RETR_EXTERNAL, cv2.CHAIN_APPROX_SIMPLE)
    
    # Draw contours on frame
    for contour in contours:
        if cv2.contourArea(contour) > 500:
            x, y, w, h = cv2.boundingRect(contour)
            cv2.rectangle(frame, (x, y), (x + w, y + h), (0, 255, 0), 2)
    
    cv2.imshow('Object Detection', frame)
    
    if cv2.waitKey(1) & 0xFF == ord('q'):
        break

cap.release()
cv2.destroyAllWindows()
```

**Why Object Detection is Important:**
- Enables robots to pick and place objects.
- Supports human-robot interaction.
- Critical for autonomous navigation.

### 3. Color-Based Robot Control

A robot that responds to different colored objects.

```python
import cv2
import numpy as np

cap = cv2.VideoCapture(0)

while True:
    ret, frame = cap.read()
    if not ret:
        break
    
    hsv = cv2.cvtColor(frame, cv2.COLOR_BGR2HSV)
    
    # Red color
    lower_red = np.array()
    upper_red = np.array()[2]
    mask_red = cv2.inRange(hsv, lower_red, upper_red)
    
    # Green color
    lower_green = np.array()
    upper_green = np.array()
    mask_green = cv2.inRange(hsv, lower_green, upper_green)
    
    red_pixels = cv2.countNonZero(mask_red)
    green_pixels = cv2.countNonZero(mask_green)
    
    if red_pixels > green_pixels:
        print("Red detected - move forward")
    elif green_pixels > red_pixels:
        print("Green detected - turn right")
    else:
        print("No color detected - stop")
    
    cv2.imshow('Frame', frame)
    
    if cv2.waitKey(1) & 0xFF == ord('q'):
        break

cap.release()
cv2.destroyAllWindows()
```

**Why Color-Based Control is Important:**
- Simple way to program robot behavior.
- Useful in sorting and assembly tasks.
- Easy to implement with basic vision.
