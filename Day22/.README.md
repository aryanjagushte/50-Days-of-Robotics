# Day 22 - Turtlesim Shape Drawing Project in ROS 2

## Overview

Day 22 turns the earlier Turtlesim practice into a small **ROS 2 project**. Instead of just moving a turtle with a simple custom node, this lesson uses Python and Turtlesim together to draw **custom shapes**, specifically a **square** and a **circle**, using **two turtles** in the simulator.

This day is valuable because it moves from basic node creation into logic-based motion design. It shows how ROS 2 topics, Python code, timing, and velocity control can be combined to create a visible project outcome rather than only a simple movement demo.

---

## What This Project Does

The project uses:
- A custom ROS 2 Python node
- Turtlesim
- Two turtle command topics
- Motion logic for shape drawing

The video explains that:
- One turtle is used to draw a **square**
- Another turtle is used to draw a **circle**

This makes the exercise more interesting than the previous single-turtle example, and it gives better practice with handling multiple topics and multiple moving entities inside the same simulation.

---

## Why Two Turtles are Used

In the previous Turtlesim lesson, only one turtle was used. In Day 22, the project expands that idea by adding a second turtle and giving each turtle a separate role.

This is useful because it teaches:
- How to control multiple simulated robots
- How each turtle has its own topic
- How one Python node can manage more than one publisher
- How project complexity increases in a structured way

That makes this lesson a strong bridge between simple practice and more project-style ROS 2 thinking.

---

## Topic Discovery with `ros2 topic list`

A practical point emphasized in the lesson is that topic names should be identified using:

```bash
ros2 topic list
```

This command helps you discover the available topics for the turtles. The video explains that this is how the command topics for both turtles are found before using them in code.

This is an important beginner habit because in ROS 2, it is better to inspect the system and verify topic names rather than assuming them from memory.

---

## Publisher Setup for Two Turtles

The custom Python node created in this project includes **two publishers**:
- one publisher for turtle 1
- one publisher for turtle 2

That is necessary because each turtle has its own velocity topic. The node publishes motion commands separately so that each turtle can follow its own shape-drawing behavior.

This is one of the most practical ideas in the lesson because it shows that a single node can coordinate multiple outputs when needed.

---

## Drawing a Square

The first shape in the project is a **square**.

The square is created by repeating a simple motion pattern:
1. Move forward in a straight line
2. Rotate by 90 degrees
3. Repeat the process until four sides are completed

The video explains that:
- linear velocity is used while drawing each side
- angular velocity is used while rotating
- the 90-degree turn is represented using approximately `1.57` radians

This is a very useful robotics lesson because it connects code directly to geometry and motion control.

---

## Motion Logic for the Square

To draw the square properly, the turtle must not only move and rotate, but also do so with correct timing.

The lesson explains that:
- the turtle first moves forward
- then turns
- then moves forward again
- and repeats this sequence until the square is completed

A short delay is also used around the turning steps so the shape becomes visually clearer and more controlled. This is an important detail because robotics motion often depends not just on velocity values, but also on **when** those values are applied.

---

## Drawing a Circle

The second shape in the project is a **circle**, and it is handled by the second turtle.

The circle is created by publishing both:
- linear velocity
- angular velocity

at the same time in a controlled way so the turtle follows a curved path continuously.

This builds directly on the earlier lesson where the turtle was made to move in a circular path, but now the motion is made more structured and project-oriented.

---

## Why the Circle Logic is Better Here

A very helpful detail from the lesson is that the circle is not made to run forever. Instead, the video uses a more controlled logic with a **for loop** so that the circle finishes after a defined number of steps.

The explanation in the transcript says the circle is created using **72 steps**, each with about **0.1 seconds**, which results in about **7.2 seconds** of movement before the turtle stops.

This is a better approach because:
- the circle gets completed cleanly
- the turtle stops automatically
- the project behaves more predictably
- the logic is more reusable

That is an important step forward from just making the turtle spin continuously.

---

## Stopping After the Shape is Done

Another useful improvement in this project is that the turtle is explicitly stopped after the shape is completed.

For the circle, the lesson mentions that after the loop finishes:
- a completion message is printed
- the turtle’s linear velocity is set to zero
- the turtle’s angular velocity is set to zero

This is an excellent robotics programming habit because motion should usually end with a clear stop command unless continuous movement is intentional.

---

## Logging and Feedback

The project includes log or print messages in the terminal to make the behavior easier to understand while the node runs.

For example, the code prints messages to indicate:
- that the turtle is moving forward
- that the circle is completed
- that turtle 2 is stopping

This is useful because it gives clear runtime feedback and helps you understand the order of operations while observing the simulator.

---

## Spawning the Second Turtle

Before the project node runs fully, a second turtle needs to be available in Turtlesim. The lesson shows that another turtle can be created using the appropriate ROS 2 command before running the custom project node.

This reinforces an important practical idea:
- if your logic expects multiple turtles
- then the simulator must actually contain multiple turtles before your node starts controlling them

That may sound obvious, but it is an important step in project setup.

---

## Running the Project Node

After setting up Turtlesim and the extra turtle, the custom node is run from the ROS 2 package. The transcript shows the node being run and then the simulator visually demonstrates the shapes being created.

When executed successfully:
- one turtle draws a square
- another turtle draws a circle
- the motion is visible in real time

This makes Day 22 one of the first lessons in the series that really feels like a complete mini-project instead of only a syntax exercise.

### Tips for Success

- Run `ros2 topic list` before writing topic names into your code.
- Start by controlling one turtle, then expand to two.
- Test square logic and circle logic separately before combining them.
- Use timing carefully, because shape accuracy depends on it.
- Add clear stop commands after finishing motion.
- Print messages in the terminal so execution flow is easy to debug.
- Try modifying the project later to draw triangles or rectangles as extra practice.
