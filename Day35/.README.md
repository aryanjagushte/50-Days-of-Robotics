# Day 35 - Creating an SSH Application in ROS 2 to Control Robot in Gazebo

## Overview

Day 35 covers **how to create an application in ROS 2** that uses **SSH (Secure Shell)** to control a robot in Gazebo simulation. The video explains that this is what most developers are fascinated by — creating ROS 2 applications that can communicate over networks using SSH.

This is an important milestone because SSH-based control allows you to test robot functionality remotely, which is essential for real-world robotics deployment where the control computer and robot may be on different machines.

SSH (Secure Shell) is important for ROS 2 applications because:

- It enables **remote communication** between computers
- It allows you to **test robot functionality remotely**
- It's essential for **real-world robot deployment**
- It lets you control robots from different machines
- It provides **secure network communication**
- It's what most developers use for distributed robotics
