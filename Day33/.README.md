# Day 33 - ROS 2 Gazebo Plugins: Moving Your Robot

## Overview

Day 33 explains how to **add Gazebo plugins to your robot model** so that it can actually move in the simulation. After spawning the robot in Gazebo on Day 32, this lesson shows how to enable **motor control via ROS 2** using Gazebo plugins, specifically the **Gazebo_Ros_Simple_Motor_Plugin**.

This is a critical step because a robot that only exists visually in Gazebo but cannot move is not useful for testing control, navigation, or autonomy. Plugins are what connect ROS 2 commands to physical motion in the simulator.

The lesson focuses on adding movement capability to the robot using Gazebo plugins. The key steps include:

- Understanding what Gazebo plugins are and why they're needed
- Installing required Gazebo ROS packages
- Adding the Simple Motor Plugin to URDF
- Configuring joint controllers
- Setting up ROS 2 control
- Testing motor movement in Gazebo
- Verifying robot can respond to commands

This makes Day 33 a practical guide for making robots move in Gazebo simulation.

## Why Gazebo Plugins are Needed

When you spawn a robot in Gazebo from URDF, it appears visually but **does not move** by default.

Gazebo plugins are required because:
- They add physics behavior to joints
- They enable motor control
- They connect ROS 2 commands to Gazebo simulation
- They allow the robot to respond to velocity commands
- They make the robot physically interactive in the simulator

Without plugins, your robot is just a visual model with no motion capability.

---

## What is Gazebo_Ros_Simple_Motor_Plugin?

The video uses the **Gazebo_Ros_Simple_Motor_Plugin** to enable robot movement.

This plugin:
- Associates with specific joints in the URDF
- Allows those joints to be controlled by ROS 2
- Enables velocity commands to move the robot
- Provides simple motor control without complex configuration
- Connects ROS 2 topics to Gazebo physics

It is a beginner-friendly plugin for getting robots to move.

## Installing Gazebo ROS Packages

Before adding plugins, you need to install the required Gazebo ROS packages.

The installation command shown:

```bash
sudo apt install ros-humble-gazebo-ros-pkgs*
```

This includes:
- Gazebo ROS integration packages
- Simple motor plugin support
- Joint controller packages
- ROS 2 control packages for Gazebo

Having these packages installed ensures the plugin can be loaded properly.

---

## Adding Plugin to URDF File

The core of the lesson is modifying the URDF file to include the Gazebo plugin.

You need to add a `<gazebo>` block in your URDF:

```xml
<gazebo reference="joint_name">
  <plugin name="gazebo_ros_simple_motor" filename="libgazebo_ros_simple_motor.so">
    <joint_name>joint_name</joint_name>
    <ros>
      <remapping>command:=joint_cmd</remapping>
    </ros>
  </plugin>
  
  <!-- Physics properties -->
  <mu1>1.0</mu1>
  <mu2>1.0</mu2>
  <fdir1>1 0 0</fdir1>
</gazebo>
```

Key elements:
- `reference` specifies which joint the plugin controls
- `plugin name` and `filename` load the simple motor plugin
- `joint_name` must match the URDF joint
- `ros remapping` sets up topic mapping for commands
- Physics properties (mu1, mu2) define friction

## Configuring Joint Controllers

The plugin works with joint controllers to move the robot.

You need to:
- Define which joints are controlled
- Set controller parameters (PID values, limits)
- Configure the controller manager
- Ensure the controller can receive ROS 2 commands

This setup allows velocity commands to be translated into joint motion.

## Setting Up ROS 2 Control

ROS 2 Control is the framework that manages robot control in ROS 2.

For Gazebo integration:
- The ROS 2 control node publishes velocity commands
- The Gazebo plugin receives these commands
- The plugin applies motion to the joints
- The robot moves in simulation

This creates the full control loop from ROS 2 to Gazebo physics.

## Testing Motor Movement

After adding the plugin and configuring everything, you test if the robot moves.

Testing steps:
- Launch Gazebo with your modified URDF
- Verify the robot appears in the world
- Send velocity commands from ROS 2
- Observe if the robot moves in Gazebo
- Check joint positions update correctly

If the robot moves, the plugin is working correctly.

---

## Publishing Velocity Commands

To move the robot, you publish velocity commands to the appropriate topic.

Common approach:
- Use `ros2 topic pub` to send commands
- Publish to the command topic configured in the plugin
- Command format is typically twist (linear and angular velocity)

Example:
```bash
ros2 topic pub /joint_cmd twist "{linear_x: 0.5, angular_z: 0.0}"
```

This sends a velocity command that the plugin translates into motion.

---

## Verifying Robot Movement

After publishing commands, verify the robot moves correctly:

- Check if the robot position changes in Gazebo
- Observe joint angles updating
- Confirm the robot responds smoothly to commands
- Test different velocity values
- Verify both forward and backward motion work

This verification ensures the control system is functioning properly.
