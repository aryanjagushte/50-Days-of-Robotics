## What is LiDAR Sensor

LiDAR (Light Detection and Ranging) is a sensor that measures distance using laser light.

### LiDAR Basics:

- **Technology**: Emits laser pulses and measures return time
- **Output**: Distance measurements in 2D or 3D
- **Range**: Typically 0.1m to 12m for robotic LiDAR
- **Accuracy**: Millimeter to centimeter precision
- **Update Rate**: 10Hz to 100Hz scanning frequency
- **Use Case**: SLAM mapping, obstacle detection, navigation

LiDAR is essential for robotics because it provides accurate distance data for mapping and navigation.

---

## Why LiDAR is Important for ROS 2

LiDAR is the primary sensor for ROS 2 robotics because:

- **SLAM Mapping**: LiDAR data builds environment maps
- **Navigation**: LiDAR detects obstacles for path planning
- **Localization**: LiDAR helps track robot position in map
- **Obstacle Avoidance**: LiDAR detects objects in robot path
- **Industry Standard**: Most mobile robots use LiDAR
- **ROS 2 Support**: Many LiDAR packages available for ROS 2

Without LiDAR, autonomous navigation is very limited.

## Connecting LiDAR to Robot Hardware

LiDAR Sensor (USB)
↓ (USB Cable)
Raspberry Pi / Microprocessor (USB Port)


### Hardware Connection:


### Connection Steps:

1. **Power LiDAR**: Use USB for 5V power (or separate 5V supply)
2. **Connect USB**: Plug LiDAR USB into Raspberry Pi
3. **Identify Port**: Check `/dev/ttyUSB0` for LiDAR device
4. **Verify Connection**: Use `ls /dev/ttyUSB*` to confirm
5. **Test Data**: Use LiDAR demo tool to verify scanning

LiDAR connects via USB serial, making it simple to integrate.

## Installing ROS 2 LiDAR Driver Packages

### Install RPLiDAR ROS 2 Package:

```bash
# For ROS 2 Humble
sudo apt install ros-humble-rplidar-ros*

# Or install from source
cd ~/mobile_robot_ws/src
git clone https://github.com/Slamtec/rplidar_ros.git
cd ~/mobile_robot_ws
colcon build
source install/setup.bash
```

This installs the driver package for RPLiDAR sensors.

## Configuring LiDAR in ROS 2 Node

### Launch LiDAR Node:

```bash
# Launch RPLiDAR node
ros2 launch rplidar_ros rplidar_s1_client.launch.py

# Or specify serial port manually
ros2 run rplidar_ros rplidar_client --serial-port /dev/ttyUSB0
```

### Configuration Parameters:

```yaml
# LiDAR configuration
serial_port: "/dev/ttyUSB0"
baud_rate: 256000
frame_id: "laser"
```

This sets up LiDAR to publish scan data.

---

## Publishing LiDAR Scan Data to ROS 2 Topics

### LiDAR Topic Output:

```bash
# LiDAR publishes to /scan topic
ros2 topic list
# Output: /scan

# View LiDAR data
ros2 topic echo /scan
```

### Message Type:

```python
# LiDAR uses LaserScan message
from sensor_msgs.msg import LaserScan

# LaserScan contains:
# - angle_min, angle_max
# - angle_increment
# - range_min, range_max
# - ranges (distance array)
# - intensities (optional)
```

LiDAR data is available on `/scan` topic for SLAM and navigation.

## Testing LiDAR Data in Rviz2

### Visualize LiDAR in Rviz2:

1. **Open Rviz2**: `rviz2`
2. **Add Display**: Click "Add" → "LaserScan"
3. **Set Topic**: Select `/scan`
4. **Set Fixed Frame**: Set to `laser` or `base_link`
5. **Verify Data**: LiDAR points should appear in 3D view

If you see LiDAR points in Rviz2, the integration works!

## Hardware Integration Best Practices

### Best Practices for LiDAR Integration:

1. **Secure Mounting**: Mount LiDAR firmly on robot to prevent vibration
2. **Power Stability**: Use stable 5V power supply for LiDAR
3. **USB Cable Quality**: Use high-quality USB cable for reliable data
4. **Cable Management**: Secure cables to prevent tangling
5. **Height Placement**: Mount LiDAR at appropriate height (avoid floor/ceiling)
6. **Clear Field**: Ensure LiDAR has clear 360° field of view
7. **Noise Reduction**: Use shielded cables to reduce electrical noise
8. **Testing**: Test LiDAR before full robot integration

Following best practices ensures reliable LiDAR performance.
