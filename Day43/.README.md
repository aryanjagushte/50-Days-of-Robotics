## Creating Python ROS 2 Node for Serial Communication

The **most easy way** is to create a Python ROS 2 node that connects serially.

### Complete ROS 2 Node Code (Python):

```python
#!/usr/bin/env python3
import serial
import rclpy
from rclpy.node import Node
from std_msgs.msg import String, Int32
from geometry_msgs.msg import Twist

class ArduinoMotorController(Node):
    def __init__(self):
        super().__init__('arduino_motor_controller')
        
        # Configuration
        self.serial_port = '/dev/ttyUSB0'  # Arduino serial port
        self.baud_rate = 9600
        
        # Initialize serial connection
        try:
            self.serial_connection = serial.Serial(
                self.serial_port, 
                self.baud_rate, 
                timeout=1
            )
            self.get_logger().info('Serial connection established')
        except Exception as e:
            self.get_logger().error(f'Serial connection failed: {e}')
            return
        
        # Create subscriber for motor commands
        self.cmd_sub = self.create_subscription(
            Twist,
            'cmd_vel',
            self.command_callback
        )
        
        # Create publisher for sensor data
        self.sensor_pub = self.create_publisher(
            Int32,
            'sensor_data',
            10
        )
        
        # Timer to read sensor data from Arduino
        self.timer = self.create_timer(0.1, self.read_sensor_callback)
        
    def command_callback(self, msg):
        """Receive motor commands from ROS 2 and send to Arduino"""
        linear = msg.linear.x
        angular = msg.angular.z
        
        # Format command for Arduino
        command = f"MOV {linear:.2f} {angular:.2f}\n"
        
        # Send to Arduino via serial
        try:
            self.serial_connection.write(command.encode())
            self.get_logger().debug(f'Sent command: {command}')
        except Exception as e:
            self.get_logger().error(f'Serial write failed: {e}')
    
    def read_sensor_callback(self):
        """Read sensor data from Arduino via serial"""
        try:
            if self.serial_connection.in_waiting > 0:
                data = self.serial_connection.readline().decode().strip()
                if data:
                    self.get_logger().debug(f'Received: {data}')
                    # Parse and publish sensor data
                    sensor_msg = Int32()
                    sensor_msg.data = int(data)
                    self.sensor_pub.publish(sensor_msg)
        except Exception as e:
            self.get_logger().error(f'Serial read failed: {e}')
    
    def destroy_node(self):
        """Close serial connection on shutdown"""
        if self.serial_connection:
            self.serial_connection.close()
        super().destroy_node()

def main(args=None):
    rclpy.init(args=args)
    node = ArduinoMotorController()
    rclpy.spin(node)
    node.destroy_node()
    rclpy.shutdown()

if __name__ == '__main__':
    main()
```

This ROS 2 node:
- Connects to Arduino via serial
- Subscribes to `cmd_vel` (ROS 2 standard velocity topic)
- Sends commands to Arduino
- Receives sensor data from Arduino
- Publishes sensor data to ROS 2

## Setting Up Serial Port Connection in ROS 2

The serial port setup is critical for Arduino connection.

### Serial Port Configuration:

```python
# Serial connection parameters
self.serial_port = '/dev/ttyUSB0'  # Change based on your system
self.baud_rate = 9600              # Must match Arduino code

# Create serial connection
self.serial_connection = serial.Serial(
    self.serial_port,
    self.baud_rate,
    timeout=1
)
```

### Finding Serial Port:

```bash
# List all serial devices
ls /dev/ttyUSB*
ls /dev/ttyACM*

# Check which port Arduino is on
dmesg | grep tty
```

Common ports:
- `/dev/ttyUSB0` - USB serial devices
- `/dev/ttyACM0` - Arduino devices
- `/dev/ttyS0` - Built-in serial

## Publishing Motor Commands from ROS 2 Node

The ROS 2 node publishes motor commands using the standard `cmd_vel` topic.

### Publishing Commands:

```python
# Subscribe to cmd_vel topic (standard ROS 2 navigation topic)
self.cmd_sub = self.create_subscription(
    Twist,
    'cmd_vel',            # Standard velocity topic
    self.command_callback
)

# In callback, format and send to Arduino
def command_callback(self, msg):
    linear = msg.linear.x   # Forward/backward speed
    angular = msg.angular.z  # Rotation speed
    
    # Format for Arduino
    command = f"MOV {linear:.2f} {angular:.2f}\n"
    self.serial_connection.write(command.encode())
```

This uses ROS 2's standard `Twist` message for velocity commands.

## Handling Serial Data Parsing in ROS 2 Node

Parsing serial data correctly is essential.

### Command Format:

```python
# Arduino expects: "MOV linear angular\n"
# Example: "MOV 0.5 0.0\n"

def command_callback(self, msg):
    linear = msg.linear.x
    angular = msg.angular.z
    command = f"MOV {linear:.2f} {angular:.2f}\n"
    self.serial_connection.write(command.encode())
```

### Sensor Data Format:

```python
# Arduino sends: sensor_value\n
# Example: "1023\n"

def read_sensor_callback(self):
    data = self.serial_connection.readline().decode().strip()
    if data:
        sensor_msg = Int32()
        sensor_msg.data = int(data)
        self.sensor_pub.publish(sensor_msg)
```

Consistent formatting ensures reliable communication.

## Debugging Serial Communication Issues

Common issues and solutions:

| Issue | Solution |
|-------|----------|
| **Port not found** | Check `ls /dev/ttyUSB*` for correct port |
| **Permission denied** | `sudo chmod 666 /dev/ttyUSB0` |
| **No data received** | Check baud rate matches (9600) |
| **Commands not working** | Verify command format matches Arduino |
| **Connection closed** | Check USB connection, restart node |
| **Timeout errors** | Increase timeout in serial setup |

Proper debugging ensures reliable communication.
