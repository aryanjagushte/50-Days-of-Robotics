# Day 15 - Creating a Custom ROS 2 Package

## Overview

Day 15 introduces one of the most important practical steps in ROS 2 development: **creating a custom package**. After understanding workspaces in Day 14, this day moves one step ahead and shows how packages are created inside the ROS 2 workspace structure.[page:6]

This lesson is short, but it is highly important because packages are the building blocks of ROS 2 projects. In future robotics applications, different functionalities such as motion control, sensor handling, vision, and navigation are often organized into separate packages, which makes the overall system structured and scalable.[page:6]

---

## From Workspace to Package

In Day 14, the focus was on understanding the ROS 2 file system and creating a workspace with a `src` folder. Day 15 builds directly on that foundation by showing that packages are created **inside the `src` folder** of the workspace.[page:6]

This relationship is important:

- **Workspace** = the overall project environment
- **src folder** = the location where packages are stored
- **Package** = an individual ROS 2 component inside the workspace[page:6]

This structure helps developers organize robotics projects properly and makes it easier to manage different modules later.[page:6]

---

## What is a ROS 2 Package?

**Simple Definition:**

A **ROS 2 package** is a structured folder that contains the files, configuration, and code needed for a specific part of a robotics project.

You can think of a package as:
- A module of your robotics project
- A container for related code and resources
- A reusable unit of development in ROS 2

For example, in a robot project, you may create separate packages for:
- Motor control
- Sensor processing
- Obstacle avoidance
- Camera handling
- Robot navigation

This modular design is one of the biggest strengths of ROS 2.

---

## Why Packages Matter

As robotics projects grow, writing everything in one place becomes difficult to manage. Packages solve this problem by separating responsibilities into organized components.[page:6]

### Benefits of Packages

- Better project organization
- Easier code maintenance
- Clear separation of functionalities
- Reusable structure
- Better scalability for large robot systems

Instead of creating one large and confusing robotics project, ROS 2 encourages dividing the project into smaller manageable packages.

## Creating a Custom Package

The video shows how to create a custom package in **ROS 2 Humble** using the terminal.[page:6]

### Important Idea

The package is created inside the `src` directory of the workspace. This is the correct and standard location for package creation in ROS 2.[page:6]

### Basic Command Style

A ROS 2 package is commonly created using the ROS 2 package creation command in the workspace source directory:

```bash
ros2 pkg create my_robot_package
```

In practice, this command is usually run after moving into the `src` folder of the workspace.

Example flow:

```bash
cd mobile_robot/src
ros2 pkg create my_robot_package
```

This creates a new custom ROS 2 package that can later be expanded with code, nodes, and other project files.

## Why Custom Packages are Important

The word **custom** is important here because ROS 2 gives you the freedom to create packages based on your own robot project needs.

This means you are not limited only to existing packages. You can build your own packages for:
- Your own robot design
- Your own software logic
- Your own algorithms
- Your own control systems

This is what makes ROS 2 flexible and useful for both learning and real-world robotics development.
