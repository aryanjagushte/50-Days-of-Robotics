## Installation Preparation

Before starting the ROS 2 installation, the first step is to make sure that the Ubuntu system is updated. This is important because outdated packages or missing dependencies can sometimes create installation problems.

### Update the System

```bash
sudo apt update
sudo apt upgrade
```

### Why This Step Matters

- Refreshes the package list.
- Upgrades outdated packages.
- Helps avoid dependency-related issues.
- Makes the system ready for ROS 2 installation.

The video also mentions that internet speed can affect how quickly these steps complete, so patience is important during package updates.

---

## Main ROS 2 Installation Flow

The Day 13 video walks through the ROS 2 installation process using the Ubuntu Debian package method from the official documentation.

### Main Installation Steps Covered

1. Set locale.
2. Enable the ROS 2 software repository.
3. Add the required GPG key.
4. Update the package list again.
5. Install ROS 2 Humble.
6. Install development tools.
7. Source the ROS 2 environment.
8. Test the installation.

This flow is important because each step prepares the system for the next one. Skipping steps can cause errors later.

---

## Setting Locale

The installation starts with setting the system locale properly.

### Why Locale Setup Matters

- Ensures the system language and encoding are configured properly.
- Helps avoid issues related to package installation and terminal output.
- Recommended as part of the official setup process.

This step may feel small, but it is a standard part of properly preparing Ubuntu for ROS 2.

---

## Setting Up Sources and Repository

Another major part of the process is adding the ROS 2 repository to the Ubuntu system. This tells Ubuntu where to fetch the ROS 2 packages from.

### Why Repository Setup Matters

- Ubuntu needs to know where ROS 2 packages are hosted.
- Without the repository, the ROS packages cannot be installed directly through `apt`.
- This step connects your system to the ROS package source.

The video also explains that some commands may look long, but they are simple if copied carefully from the official documentation.

---

## Setting Up the GPG Key

The ROS 2 repository uses a **GPG key** for package authentication. This key helps Ubuntu verify that the packages are coming from a trusted source.

### Why the GPG Key Matters

- Confirms package authenticity.
- Helps secure the installation process.
- Required for properly using the ROS repository.

The video specifically mentions that if an error happens during this step, you should go back and try the previous setup steps again carefully, because some issues are often solved by redoing the earlier commands properly.

---

## Updating Again After Repository Setup

Once the repository and GPG key are added, the system package list needs to be updated again.

```bash
sudo apt update
sudo apt upgrade
```

### Why This Second Update is Needed

- Refreshes Ubuntu so it now includes the ROS 2 repository packages.
- Ensures the system can detect newly available ROS 2 packages.
- Prepares for the actual ROS installation step.

This is an important detail because many beginners forget that after adding a new repository, another update is needed.

---

## Installing ROS 2 Humble

The main installation step in the video is installing the **desktop version** of ROS 2 Humble.

### Recommended Installation

```bash
sudo apt install ros-humble-desktop
```

### Why the Desktop Version is Recommended

The video clearly recommends installing the **desktop** version instead of only the base version.

**Desktop version includes:**
- A more complete ROS 2 setup
- Graphical tools
- Visualization tools
- A better environment for learning and development

**Base version is more limited:**
- Does not include some GUI-related tools
- Better suited for minimal systems or more advanced custom setups

For beginners in this challenge, the **desktop version** is the right choice because it gives a fuller ROS experience from the start.

## Sourcing the ROS 2 Environment

After installation, the ROS 2 environment must be sourced in the terminal.

### Command

```bash
source /opt/ros/humble/setup.bash
```

### Why Sourcing is Important

ROS 2 gets installed in the `/opt/ros/humble/` directory. Sourcing the setup file tells the current terminal session where ROS 2 is installed and makes ROS commands available.

**Important beginner point:**
- Sourcing only affects the current terminal.
- If you open a new terminal, you need to source it again unless you add it to your shell configuration later.

This is one of the most important things to understand early in ROS 2.
