## Autonomous Mobile Robot Complete Breakdown

The video provides a **complete breakdown of the autonomous mobile robot in an easy way**.

### Robot Components:

| Component | Purpose | Connection |
|-----------|---------|------------|
| **Microprocessor** | Runs ROS 2, navigation, SLAM | Raspberry Pi 4/5 or Jetson |
| **Microcontroller** | Motor control, sensor reading | Arduino Uno |
| **Motor Driver** | Controls DC motors | L298N or TB6600 |
| **DC Motors** | Robot movement | Connected to motor driver |
| **Lidar Sensor** | SLAM mapping, navigation | USB to Raspberry Pi |
| **Camera** | Visual perception | USB/CSI to Raspberry Pi |
| **IMU** | Orientation, localization | I2C to Arduino |
| **Battery** | Power supply | Motor driver + Raspberry Pi |
| **Chassis** | Robot frame | Mounts all components |

This architecture enables autonomous navigation.

## ROS 2 Configuration for Hardware

After hardware connections, configure ROS 2.

### ROS 2 Setup Steps:

1. **Install ROS 2 Humble** on Raspberry Pi
2. **Install required packages**:
   ```bash
   sudo apt install ros-humble-slam-toolbox*
   sudo apt install ros-humble-nav2-bringup*
   sudo apt install ros-humble-rplidar-ros*
   ```
3. **Create robot description package** with URDF
4. **Configure slam_toolbox** for lidar SLAM
5. **Configure nav2** for navigation stack
6. **Create Arduino motor controller node**
7. **Test SLAM mapping** with lidar
8. **Test navigation** with goal publishing

This sets up ROS 2 for autonomous navigation.

---

## Testing Autonomous Navigation

After all connections and configuration, test the robot.

### Testing Steps:

1. **Power on robot**: Connect battery
2. **Start ROS 2**: Source workspace on Raspberry Pi
   ```bash
   source /opt/ros/humble/setup.bash
   ```
3. **Launch SLAM**:
   ```bash
   ros2 launch slam_toolbox online_async_launch.py
   ```
4. **Launch Navigation**:
   ```bash
   ros2 launch nav2_bringup navigation_launch.py
   ```
5. **Publish Goal**:
   ```bash
   ros2 action send_goal /move_base_flex nav2_msgs/action/NavigateToPose
   ```
6. **Verify Robot Moves**: Observe robot navigating
7. **Check Map**: Verify SLAM mapping in Rviz2
8. **Monitor Logs**: Check ROS 2 node output

If robot navigates autonomously, the system works!
